# EntropyShield

**Язык / Language:**  <a href="https://github.com/loanelly/EntropyShield/blob/main/README.md"><img src="img/Flags/EN1.png" height="21" align="center"/></a> <a href="https://github.com/loanelly/EntropyShield/blob/main/README_RU.md"><img src="img/Flags/result.png" height="26" align="center"/></a>

Профессиональный аппаратный **генератор истинно случайных чисел (TRNG)**, извлекающий чистую, непредсказуемую физическую энтропию из теплового/дробового шума (лавинного пробоя в обратно-смещенном p-n переходе транзистора).

Система включает в себя встроенную прошивку микроконтроллера со встроенным криптографическим **экстрактором Фон Неймана** для устранения аппаратных перекосов, а также защищенный Python-клиент для безопасной передачи сгенерированных криптографических ключей (например, **AES-256**) в конечные приложения на ПК.

---

## ⚡ Ключевые особенности

- **Квантовый источник энтропии:** Захват сырого лавинного шума с p-n перехода реального транзистора — полная неуязвимость к математическому прогнозированию или ИИ/ML анализу.
- **Очистка данных по Фон Нейману:** Потоковая фильтрация битов в реальном времени гарантирует идеальное статистическое распределение нулей `0` и единиц `1` (ровно 50/50).
- **Два режима работы:** Директивы препроцессора позволяют в один клик переключаться между браузерной **симуляцией в Wokwi** и высокоскоростным чтением **реального железа**.
- **Стабильный клиентский софт:** Скрипт на Python защищен от повреждения потока данных в Serial-порту, автоматически очищает буферы ввода и безопасно обрабатывает любые сбои связи.

## 🔬 Принцип работы (Краткий обзор)

1. **Этап Хаоса:** На физический транзистор (например, BC547) подается обратное напряжение, вызывая непредсказуемый лавинный пробой носителей заряда.
2. **Этап Усиления:** Операционный усилитель (LM358) раскачивает микровольтовые колебания шума до логических уровней 0В–5В.
3. **Этап Очистки:** Arduino считывает младший значащий бит (LSB) АЦП. **Фильтр Фон Неймана** проверяет входящие биты парами:
   - `10` → Превращается в чистую `1`
   - `01` → Превращается в чистый `0`
   - `00` или `11` → Мгновенно сбрасываются для уничтожения аппаратных наводок и сетевого гула 50 Гц.
4. **Этап Сборки:** Скрипт на Python считывает очищенный поток, валидирует байты по шестнадцатеричной матрице и собирает их в итоговый ключ.

## 🚀 Руководство по установке и настройке

### 1. Интерактивное тестирование в браузере (Без покупки деталей)
Чтобы мгновенно проверить работу алгоритма без сборки схемы:
1. Откройте прямую ссылку из файла **[src/documentation/wokwi_simulation.txt](src/documentation/wokwi_simulation.txt)**.
2. Убедитесь, что в самом верху кода прошивки активна строка `#define WOKWI_EMULATION_MODE true`.
3. Нажмите кнопку **Start Simulation** ⏱️. Откройте окно *Serial Monitor* в правом нижнем углу, чтобы увидеть живой поток случайных HEX-байтов.

### 2. Прошивка микроконтроллера
Для развертывания проекта на реальном чипе:
1. Подключите вашу плату (Arduino Nano / Uno / Pro Micro) к компьютеру через USB.
2. Откройте файл **[src/arduino/entropy_shield.ino](src/arduino/entropy_shield.ino)** в Arduino IDE или PlatformIO.
3. Переведите флаг работы в режим реального железа (production):
   ```cpp
   #define WOKWI_EMULATION_MODE false
   ```
4. Выберите вашу модель платы, COM-порт и нажмите кнопку **Загрузка (Upload)**.

### 3. Настройка криптографического Python-клиента
Скрипт на ПК отвечает за безопасный сбор энтропии с системной шины.

1. Перейдите в корневую папку проекта на ПК и установите библиотеку для работы с портами:
   ```bash
   pip install pyserial
   ```
2. Откройте файл **[src/python/main.py](src/python/main.py)** и убедитесь, что переменная `SERIAL_PORT` соответствует вашей ОС:
   - **Windows:** `'COM3'`, `'COM4'` и т.д.
   - **Linux / macOS:** `'/dev/ttyUSB0'`, `'/dev/ttyACM0'`
3. Запустите генератор для сбора защищенного 256-битного ключа:
   ```bash
   python src/python/main.py
   ```

## 🧱 Физическая сборка (Hardware Mode)

При переходе к реальной сборке (`WOKWI_EMULATION_MODE false`) аппаратному уровню требуется изолированное усиление. 

Ниже представлена принципиальная схема структуры. Точные номиналы резисторов и параметры фильтрующих конденсаторов указаны в файле **[src/documentation/connection_guide.txt](src/documentation/connection_guide.txt)**.

```text
 [Линия 12В/9В] ───[100кОм]───┐
                               │
                         (BC547 NPN) ───[Коллектор свободен 💨]
                               │
    [GND] ─────────────────────┴───[100нФ Конд]───► [Усилитель LM358] ───► Arduino [A0]
```
