1. ВВЕДЕНИЕ И АКТУАЛЬНОСТЬ
--------------------------
Современные компьютеры являются детерминированными автоматами. Они не способны 
создавать истинную случайность. Стандартные программные функции (например, rand() 
в C++ или random в Python) используют псевдослучайные генераторы (PRNG). Они 
работают по математическим формулам: зная начальное число (seed), злоумышленник 
или ИИ-алгоритм может полностью предсказать всю последовательность ключей.

Проект LNL-EntropyShield решает эту уязвимость. Он извлекает энтропию из 
физического мира (квантовый хаос полупроводников), делая создаваемые ключи 
абсолютно непредсказуемыми и математически невзламываемыми.

2. АППАРАТНАЯ АРХИТЕКТУРА (HARDWARE)
------------------------------------
В физической реализации (Production Mode) устройство состоит из трех блоков:

А) Источник энтропии (Шум):
   Используется обратное смещение p-n перехода NPN-транзистора (например, BC547). 
   При подаче напряжения выше пробойного, переход переходит в режим лавинного 
   пробоя. Движение электронов в этом режиме рождает тепловой и дробовой шум. 
   Этот шум имеет квантовую природу и принципиально непредсказуем.

Б) Усилительный каскад:
   Амплитуда сырого шума составляет единицы микровольт. Сигнал пропускается через 
   низкошумящий операционный усилитель (например, LM358), который раскачивает 
   амплитуду до диапазона 0-5 Вольт, необходимого для работы цифровой логики.

В) Обработка (Микроконтроллер):
   Плата Arduino Nano считывает усиленный аналоговый сигнал через встроенный 
   аналогово-цифровой преобразователь (АЦП) на пине A0. Для генерации берется 
   только самый младший, нестабильный бит (LSB) каждого замера.

3. МАТЕМАТИЧЕСКАЯ ОЧИСТКА: ЭКСТРАКТОР ФОН НЕЙМАНА
--------------------------------------------------
Любое физическое железо имеет несовершенства (наводки от электросети 50/60 Гц, 
температурный дрейф). Из-за этого в сырых битах АЦП возникает аппаратный перекос 
(например, единиц генерируется 52%, а нулей — 48%).

Для очистки потока в прошивку встроен цифровой экстрактор Фон Неймана. 
Он обрабатывает поступающие биты строго парами:
- Пары '1' и '0' (10) -> преобразуются в чистый бит '1'
- Пары '0' и '1' (01) -> преобразуются в чистый бит '0'
- Пары '00' и '11'  -> полностью отбрасываются

Этот алгоритм гарантирует идеальный баланс нулей и единиц (ровно 50/50) в 
итоговом криптографическом ключе, уничтожая любые закономерности.

4. РЕЖИМЫ РАБОТЫ (WOKWI VS PRODUCTION)
--------------------------------------
Для удобства тестирования в код встроен программный переключатель:

1) #define WOKWI_EMULATION_MODE true (Режим симуляции)
   Цифровые симуляторы (Wokwi) не умеют моделировать хаос электронов. Чтобы 
   алгоритм Фон Неймана не зависал в браузере, этот режим подмешивает 
   микросекундный джиттер (сдвиг фазы процессора) и программный PRNG. Это 
   позволяет визуально протестировать передачу данных.

2) #define WOKWI_EMULATION_MODE false (Режим реального железа)
   Все программные костыли отключаются. Процессор АЦП переводится в ускоренный 
   режим чтения. Система полностью полагается на аппаратные шумы пина A0.

5. ПРОГРАММНОЕ ОБЕСПЕЧЕНИЕ (PYTHON CLIENT)
------------------------------------------
Скрипт на стороне ПК работает как сборщик энтропии:
1. Подключается к плате по протоколу Serial (USB) на скорости 115200 бод.
2. Побайтово считывает очищенные HEX-данные.
3. Аккумулирует их до достижения нужной длины (например, 32 байта для AES-256).
4. Передает готовый ключ в целевые приложения (менеджеры паролей, шифраторы).
